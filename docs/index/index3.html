Merge Sort

We implement merge sort using recurssion. Using the left most element and right most element, we calculate the midpoint until we can no longer keep going. We perform merge sort on the following list of elements

int[] array = {9, 3, 7, 5, 6, 4, 8, 2, 10, 12};

Results

We do not return anything, rather, we perform merge sort in place.

Merging

Using the left most element, the middle, and the right most element, we split the passed array into two arrays.

static void Merge(int[] arr, int l, int m, int r){
  int[] L = new int [m];
  int[] R = new int[(r-m) + 1];
}

We then assign all repsecitve elements into each list

int i = 0;
int j = 0;
int k = 0;

for(i = 0; i < m; i++){
  L[i] = arr[i];
}
for(j = 0; j < (r-m+1); j++){
  R[j] = arr[m+j];
}
i = 0;
j = 0;

We need to compare each element in each list to eachother. Therefore, we will iterate until both lists are compared. Example:

A = {2, 5, 6}
B = {3, 1}
C = {}

Since i, j = 0, A[i] = 2 and B[i] = 3. 2 < 3, so we add 2 to set C and increment i.

A = {2, 5, 6}
B = {3, 1}
C = {2}

Again: 3 < 5, so we increment j and add 3 to set C.

A = {2, 5, 6}
B = {3, 1}
C = {2, 3}

1 < 5, so we add 1 to set C and increment j.

A = {2, 5, 6}
B = {3, 1}
C = {2, 3}

We have no more elements to compare with A, so we default and add the remaining elements.

A = {2, 5, 6}
B = {3, 1}
C = {2, 3, 5, 6}

Notice that 5 and 6 are in order. This is not always the case, but recurssion will help us fix that. Now let's see the rest in code (merging only):

while(i < m && j < (r-m+1)){
  if(L[i] < R[j]){
   arr[k++] = L[i++];
  }
  else{
   arr[k++] = R[j++];
  }
}
while(i < L.length){
  arr[k++] = L[i++];
}
while(j < R.Length){
  arr[k++] = R[j++];
}

Merge Sort

Now that we are able to merge elements we can create merge sort. This is pretty straight forward. We simply continue to call the function itself until both halves are sorted. You can view the full source code in the repo linked below. Here is how the recursive process looks like (pseudo):

MergeSort(int[] arr, int l, int r){
  Calculate midpoint;
  MergeSort(arr, l, mid);
  MergeSort(arr, mid+1, r);
  Merge(arr, l, mid, r);
}

Complexities

  • Best Case: O(n log n), since we are splitting the list into halves recursively
  • Worst case: O(n log n)
  • Average case: O(n log n)
  • Space complexity: O(n)
